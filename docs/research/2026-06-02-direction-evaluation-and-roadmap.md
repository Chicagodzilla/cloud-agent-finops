# 项目方向评估矩阵与落地路线

日期：2026-06-02

## 1. 评估维度

本评估使用 1-5 分制：

- 市场痛点：问题是否真实、高频、愿意付费。
- 项目契合度：和当前 Cloud Agent 代码、数据、智能体结构是否贴合。
- MVP 难度：越容易做出可演示版本分数越高。
- 差异化：是否避开通用客服红海，有自己的特点。
- 商业化潜力：是否容易变成项目、产品、课程或企业 PoC。
- 风险可控：数据、合规、误操作、成本等风险是否可管理。

## 3. 推荐主线

### 主线名称

Cloud Agent for FinOps & Support

### 产品定位

一个面向云服务客户、云代理商和 SaaS 运维团队的智能客服智能体，能回答产品问题、查询账单资源、诊断成本浪费、生成优化建议，并提供可审计的智能体运行记录。

### 核心卖点

- 不只是问答，而是能查资源、看账单、做建议。
- 不只是客服，而是把售前、售后、成本优化串起来。
- 不只是 demo，而是有 Trace、评估和 ROI 指标。
- 不追求全自动执行，高风险动作前必须人工确认。

## 4. 三个月落地路线

### 第 1 阶段：业务闭环 MVP（第 1-3 周）

目标：让系统能完整演示“云服务咨询 + 账单查询 + 成本优化建议”。

建议功能：

- 修正环境配置和启动脚本，提供一键启动。
- 完善云资源 mock API：实例列表、订单记录、监控指标、费用明细。
- 强化 FinOpsAgent：从资源数据生成降本建议。
- 增加“优化报告”输出：按资源、问题、建议、风险、预计收益组织。
- 前端增加结果结构化展示：资源表、建议卡片、风险提示。

验收标准：

- 用户问“这个月账单为什么贵”，系统能查到费用构成。
- 用户问“哪些服务器可以省钱”，系统能输出具体实例建议。
- 每条建议能解释依据和风险。

### 第 2 阶段：可信与可运营（第 4-7 周）

目标：解决企业不敢用、不知道效果如何的问题。

建议功能：

- 对话 Trace：记录 Orchestrator 路由、agent、工具调用、耗时和结果。
- 答案来源：每次回答必须记录引用文档或工具来源。
- 用户反馈：点赞、点踩、问题未解决、要求人工。
- 基础指标面板：自动解决率、缓存命中率、工具失败率、平均响应时长。
- Golden Set 评测：维护一组固定测试问题，比较不同 prompt 或模型效果。

验收标准：

- 每次回答都能回放完整链路。
- 能看到一周内失败最多的问题类型。
- 能用固定问题集验证修改是否让效果变差。

### 第 3 阶段：知识库飞轮（第 8-12 周）

目标：让系统从使用中持续变聪明。

建议功能：

- 未命中问题聚类。
- 从低分会话生成知识库补充建议。
- 文档冲突检测：同一主题多个答案不一致时提示人工。
- 新知识上线前评测。
- 客服人员审核工作台。

验收标准：

- 系统能每周产出“知识缺口报告”。
- 运营人员可以基于建议更新文档。
- 更新后能通过 golden set 验证效果。

## 5. Demo 场景设计

### Demo 1：成本异常诊断

用户输入：

> 这个月账单怎么比上个月贵了这么多？帮我看看哪里能省钱。

系统流程：

1. Orchestrator 识别 FinOps 意图。
2. BillingAgent 查询订单和资源费用。
3. FinOpsAgent 查询监控指标。
4. 系统找出低利用率实例和高带宽费用。
5. 输出降配、关停、购买优惠、保留实例等建议。
6. 生成报告。

展示重点：

- 多智能体协作。
- 工具调用。
- 结构化业务结果。
- 可解释建议。

### Demo 2：产品选型推荐

用户输入：

> 我是 Java 接口服务 + MySQL，预计 3000 日活，预算有限，推荐什么 ECS 和 RDS？

系统流程：

1. RecommendationAgent 收集场景。
2. ProductAgent 查询规格和产品知识。
3. 系统输出主推荐和低成本替代方案。
4. PromotionAgent 匹配活动或优惠。

展示重点：

- 售前推荐。
- 结合知识库和业务规则。
- 从客服走向转化。

### Demo 3：知识库缺口发现

用户输入：

> 最近哪些问题 AI 没回答好？

系统流程：

1. 系统读取历史对话和反馈。
2. 聚类未解决问题。
3. 标记缺失文档和冲突文档。
4. 生成 FAQ 草稿。

展示重点：

- 智能客服运营化。
- 数据飞轮。
- 不只是聊天界面。

## 6. 关键产品指标

### 业务指标

- 自动解决率。
- 人工转接率。
- 平均首响时间。
- 平均解决时长。
- 成本优化建议采纳率。
- 预计节省金额。
- 售前推荐转化率。

### 智能体质量指标

- 路由准确率。
- 工具调用成功率。
- 知识库命中率。
- 答案来源覆盖率。
- 用户追问率。
- 幻觉/无依据回答率。
- 回归评测通过率。

### 工程指标

- 平均响应耗时。
- token 成本。
- 语义缓存命中率。
- 外部 API 失败率。
- 高风险动作拦截率。

## 7. 数据与技术补强建议

### 数据侧

- 增加 mock 账单：月度账单、产品线费用、异常增长原因。
- 增加 mock 监控：CPU、内存、带宽、磁盘、连接数。
- 增加 mock 工单：问题、处理步骤、最终解决方案。
- 增加优惠活动数据：活动适用产品、时间、限制条件。
- 增加 FAQ 标注：标准答案、来源、过期时间、业务负责人。

### 架构侧

- 给每次请求生成 trace_id。
- 记录 agent routing decision。
- 记录 tool_call 参数、结果、耗时、错误。
- 给回答附带 source metadata。
- 引入高风险动作确认机制。
- 把语义缓存从“命中即返回”扩展为“命中 + 置信度 + 可回放来源”。

### 前端侧

- 增加资源表格和成本卡片。
- 增加报告视图。
- 增加 trace 侧栏。
- 增加用户反馈按钮。
- 增加知识库缺口列表。

## 8. 最小可行版本清单

第一版只建议做这些：

1. 云产品问答保留现有能力。
2. mock 账单和资源查询可用。
3. FinOpsAgent 输出结构化建议。
4. 前端展示“建议卡片 + 风险提示 + 报告”。
5. 后端记录 trace 日志。
6. 提供 10 个 demo 问题和固定评测结果。

暂时不要做：

- 真正执行云资源变更。
- 全渠道客服。
- 复杂 CRM。
- 语音客服。
- 多租户商业后台。
